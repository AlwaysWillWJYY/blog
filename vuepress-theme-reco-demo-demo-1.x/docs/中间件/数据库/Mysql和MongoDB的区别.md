### MySQL和MongoDB的区别？

MongoDB：非关系型数据库，文档型数据库

存储方式：虚拟内存+持久化

查询语句：MongoDB特殊的查询方式

适用场景：事件的记录，内容管理或者博客平台等

架构特点：可以通过副本集，以及分片来实现高可用

数据处理：数据是存储在硬盘上的，经常需要读取的数据会被加载到内存中，将数据存储在内存中，从而达到高速的读写。

MongoDB是新兴的数据库，成熟度较低

`将热数据存储在内存中，使得热数据的读取速度比较快`

MySQL：成熟的关系型数据库

不同的存储引擎对应着不同的存储方法

数据的读写都有一套成熟的SQL语句完成

缺点是在海量数据时，处理速度会变慢
